##### One.对Git的理解
所有Git教程都会介绍Git的优点，下面引述Git官网的话:[^blabala] 
而我个人对Git的理解，Git是个版本控制器，就是Git可以清晰记录每一个人对文档的修改，而且对个人而言，可以在没网时在本地进行编辑，只需要在有网时提交一下便可。


而Git绝对十分优秀，因为谷歌脸书微软推特网飞等这些耳熟能详的大公司都选择使用Git，这里要向天才Linus Torvalds致敬（据说只用了10天左右就完成了这个伟大的创造）。
##### Two.学习Git的经历及实践
在安装了Git之后，我开始按照菜鸟教程和廖雪峰教程在Git上进行实际操作，刚刚开始就出现了问题,先是didn't match any files 又出现  untracked files,没提交成功就去建立分支导致失败等。这些问题都通过请教别人和百度解决，愈挫愈勇，失败是成功之母，这次失败能加深我的印象，让我之后不犯错。现在我已经明白了git的一些概念及基本的git指令，

>概念：
>>4个区
>>>* 工作区(Working Area)
>>>* 暂存区(Stage)
>>>* 本地仓库(Local Repository)  
>>>* 远程仓库(Remote Repository）  

>>5种状态
>>>* 未修改（origin）
>>>* 已修改(modified)
>>>* 已暂存(staged)
>>>* 已提交(commited)
>>>* 已推送(pushed)

>指令：  
>>>* 配置 git config --global user.name "name"  git config --global user.email "email@email.com"
>>>* 查看 cat ~/.gitconfig  会得到  name = SparksWLL
          email = 2418610836@qq.com  的内容 
>>>* 创建文件夹 mkdir filename
>>>* 创建文件 touch name 
>>>* 创建并编辑文件 vi name   
>>>* 进入目录 cd filename  
>>>* 初始化一个git仓库 git init  
>>>* 全部文件添加到暂存区 git add
>>>* 全部文件从暂存区提交到本地仓库 git commit  
>>>* 添加指定文件到暂存区 git add filename  
>>>* 查看文件 cat 
>>>* 把文件夹里的文件列出来 ll : ls list  
>>>* 创建分支 git branch name
>>>* 创建并切换分支 git checkout -b name
>>>* 查看分支 git branch   
>>>* 最后，感觉git很智能。场景如下：  
git: 'checkou' is not a git command. See 'git --help'.
  The most similar command is
      checkout
